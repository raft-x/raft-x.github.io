What is Extended Raft?

Extended Raft is an enhancement to the Raft consensus algorithm that introduces witness servers and replication subterms to reduce infrastructure costs by minimizing the number of full replicas needed. This variant maintains Raft's strong consistency guarantees while enabling cost-effective deployments where running multiple full-storage servers may be prohibitively expensive.

Witness Servers

Lightweight servers that participate in leader election and quorum formation without storing full log data, reducing storage overhead while maintaining consensus safety.

Replication Subterms

Fine-grained replication tracking within terms enables optimized witness interactions and more efficient replication patterns.

Shortcut Replication

After initial witness acknowledgment in a subterm, subsequent entries can skip witness communication, reducing network overhead.

Safety Preserved

Maintains all of Raft's core safety properties including leader election safety, log matching, and state machine safety guarantees.

Key Capabilities

  • Dynamic Replication Sets: Leaders can adjust their replication set to include witness servers when regular servers are unavailable
  • Witness Voting Rules: Witnesses vote based on log metadata (term and subterm) rather than full log contents, enabling informed decisions with minimal storage
  • Subquorum Requirements: Candidates must obtain votes from a subquorum of regular servers before requesting witness votes, ensuring safety
  • Network Partition Tolerance: Maintains availability with a witness and a single regular server, even when other servers are unreachable
  • Optimized Witness Traffic: Shortcut replication reduces redundant witness communication after the first acknowledgment in each subterm

Research & Resources

Extended Raft builds upon the original Raft consensus algorithm described in the paper "In Search of an Understandable Consensus Algorithm" by Diego Ongaro and John Ousterhout. This extension explores additional optimization opportunities while maintaining Raft's core principles of understandability and safety.

See the Extended Raft specification document and formal TLA+ proof for a detailed description of the algorithm, its safety proofs, and formal verification.

For more information about the original Raft algorithm, visit raft.github.io. Watch this KubeCon session to learn how Extended Raft enables 2-node Kubernetes deployments in production environments.
